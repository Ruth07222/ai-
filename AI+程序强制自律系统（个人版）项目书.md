# AI+程序强制自律系统（个人版）项目书

# 一、项目概述

## 1.1 项目背景

用户存在典型的“游戏日活积极、学习任务拖延”的矛盾，核心痛点在于：个人制定的自律规则易“监守自盗”，现有锁机APP、手动监督等方式存在“可控性差、易绕过、无智能判断”的缺陷，无法达到游戏官方系统“规则固定、强制执行、不可篡改”的可靠性。用户希望构建一套结合AI智能审核与程序强制锁死的自律系统，模拟游戏官方机制，实现“学习完成自动解锁娱乐、未完成强制限制”，全程无需人工干预、不依赖他人监督，杜绝自身作弊可能。

## 1.2 项目核心目标

- 智能审核：通过AI识别学习成果真实性，判断用户是否真正完成学习（防敷衍、防摆拍、防抄录），并通过出题测验验证理解程度。

- 强制闭环：程序自动控制娱乐权限，AI审核通过则自动解锁，娱乐时间结束后自动收回权限；未通过则强制限制，用户无法手动绕过。

- 无感体验：正常完成学习后，娱乐解锁全程无感，不增加额外操作成本；未完成时，通过高摩擦设计（动态乱码密码等）冷却冲动，避免监守自盗。

- 多端适配：覆盖Windows（核心强制端）、Android（辅助强制端）、iOS（曲线救国端），实现多设备同步管控，贴合用户日常使用场景。

## 1.3 项目核心构想

以“AI为审核核心、程序为执行载体、多端协同为支撑”，构建一套“学习提交→AI审核→出题验证→权限解锁→时间回收”的全自动闭环系统，模拟游戏官方服务器的“规则固定、强制执行、数据不可篡改”特性，将用户从“被动自律”转化为“系统强制下的主动执行”，彻底解决个人自律中的“放水、找借口、监守自盗”问题。

# 二、项目雏形（整体架构）

## 2.1 整体架构图（核心逻辑）

用户学习 → 提交学习成果（文字/图片）→ 程序接收并上传至AI审核模块 → AI完成“真实性判断+出题测验” → 用户答题 → AI判分（≥2题正确为通过） → 程序接收审核结果 → 已通过：自动解锁娱乐权限（设定时长）→ 娱乐时长结束：程序自动收回权限，重新锁死 → 未通过：保持锁死状态，触发高摩擦拦截（如动态乱码密码）。

## 2.2 核心模块划分

1. 学习提交模块：用户上传学习成果（笔记、习题、推导过程等）的入口，支持文字、图片上传，适配多端操作。

2. AI审核模块：系统核心，调用大模型API（优先GPT-4/Gemini，备选国内模型），按固定规则完成“学习真实性审核、出题、判分”，输出标准化结果。

3. 权限控制模块（核心执行端）：分三端实现，负责接收AI审核结果，执行“解锁娱乐”或“强制锁死”操作，确保不可绕过。

4. 动态密码模块：iOS端核心辅助模块，AI审核未通过时生成一次性32位乱码密码，增加解锁摩擦，冷却用户冲动。

5. 状态同步模块：通过云端变量/本地变量，同步多端“授权状态”（已授权/未授权），确保多设备管控一致。

## 2.3 核心设计原则

- 强制不可绕过：程序执行权高于用户主观操作，通过系统级权限、物理/软件限制，杜绝用户手动解锁、杀进程、卸载等作弊行为。

- AI绝对中立：AI按固定规则审核，不放水、不闲聊、不情绪化，完全模拟游戏官方系统的“铁面无私”。

- 无感与高摩擦结合：完成学习后全程无感，未完成时触发高摩擦拦截，平衡用户体验与自律效果。

- 低成本落地：个人可开发，无需企业证书、无需越狱/root、无需复杂服务器，依托现有工具和API快速实现。

# 三、各系统解决方案（分端落地）

## 3.1 核心强制端：Windows电脑端（最稳、最强强制）

### 3.1.1 权限获取（个人可实现）

无需企业证书、无需微软认证，个人可通过管理员权限+系统API获取系统级管控权限，核心两种方案：

1. 方案A：管理员权限+系统API拦截（基础版）
        

    - 技术：Python + pywin32/ctypes，调用Windows API（LockWorkStation()）实现强制锁屏，搭配全局输入屏蔽。

    - 权限：普通用户权限可实现基础锁屏，管理员权限可屏蔽Ctrl+Alt+Del、任务管理器、Win键等快捷键，防止绕过。

    - 能力：强制锁屏、拦截所有退出/解锁快捷键，只有AI审核通过后，程序自动解除锁屏。

2. 方案B：Kiosk模式（分配的访问，最强版）
        

    - 技术：利用Windows内置“分配的访问”功能，通过Python/C#配置本地账户，实现单APP锁定。

    - 权限：管理员权限+本地账户配置，无需额外资质。

    - 能力：账户登录后仅能运行本自律程序，屏蔽开始菜单、任务栏、任务管理器，重启后依然保持锁定，用户无法手动退出，仅能通过AI审核解锁。

### 3.1.2 核心功能实现

- 学习提交：支持文字输入、图片上传（拍照/截图），程序自动接收并上传至AI审核模块。

- AI联动：调用GPT-4/Gemini API，传入固定审核指令，接收AI返回的“审核结果、题目、答案”，自动判分。

- 权限控制：审核通过 → 自动解锁娱乐APP（解除进程限制），并启动后台计时；计时结束 → 自动调用系统API锁屏/重新限制娱乐APP，强制收回权限。

- 防绕过设计：管理员权限运行、屏蔽快捷键、防杀进程、开机自启，确保用户无法手动绕过程序控制。

## 3.2 辅助强制端：Android手机端（强强制，个人可落地）

### 3.2.1 权限获取（个人可实现）

核心采用“设备管理员权限（Device Admin）”，无需企业证书、无需root，个人可直接申请激活：

1. 权限申请流程：APP清单注册DeviceAdminReceiver，声明BIND_DEVICE_ADMIN权限 → 跳转系统设置页，用户手动激活设备管理员（不可静默） → 激活后获得系统级管控权限。

2. 核心权限：强制锁屏（lockNow()）、禁用相机/安装应用/USB调试、设密码策略、防止APP被强制停止/普通卸载。

### 3.2.2 核心功能实现

- 学习提交：同Windows端，支持文字、图片上传，同步至AI审核模块。

- AI联动：接收AI审核结果，同步多端授权状态。

- 权限控制：审核通过 → 自动解锁娱乐APP，解除锁屏限制；未通过 → 调用lockNow()强制锁屏，禁用娱乐APP；娱乐时长结束 → 自动锁屏，重新限制。

- 防绕过设计：设备管理员激活后，需先在系统设置中取消激活才能卸载APP（可将取消步骤藏深+添加验证），后台服务前台保活，防止被清后台。

## 3.3 曲线救国端：iOS手机端（无系统级权限，高摩擦拦截）

### 3.3.1 权限现状与核心思路

iOS系统封闭，个人开发者无法获取系统级强制锁死权限，核心思路：不强行锁死系统，而是通过“高摩擦密码+快捷指令自动化+状态联动”，增加用户破戒成本，冷却冲动，实现“心理级强制”。

### 3.3.2 核心功能实现（个人可落地）

1. 状态机配置：利用iOS快捷指令的“iCloud同步变量”，创建study_authorization变量（取值：allowed/denied），同步多端授权状态。

2. 自动化触发：创建个人自动化，勾选目标娱乐APP，触发条件设为“已打开”，关闭“运行前询问”，实现无感触发。

3. 条件分支执行：


    - 已授权（AI审核通过）：快捷指令无操作，直接放行，娱乐APP正常打开，全程无感。

    - 未授权（AI审核未通过）：弹出32位动态乱码密码输入框，需手动逐字输入，输错一位则清空重来并强制锁屏；密码为一次性、自动更新，仅AI通过后才由服务器生成并显示。

4. AI联动：通过快捷指令调用云端服务，查询AI审核结果，自动更新study_authorization变量；未完成学习或娱乐超时，自动重置为denied。

5. 双重保险：配合iOS自带“屏幕使用时间”，设置娱乐APP限额为0，密码设为超长乱码（由服务器生成），进一步强化限制。

### 3.3.3 动态密码规则（防监守自盗核心）

- 密码规格：32位，包含10位大写字母+10位小写字母+6位数字+6位符号，随机打乱，无规律。

- 生成逻辑：由云端服务（Python后端）随机生成，每次AI审核通过后生成新密码，一次性使用，10分钟超时失效。

- 输入限制：不可复制粘贴，必须手动逐字输入，输错即清空重来，叠加锁屏操作，最大化摩擦成本。

# 四、AI审核模块解决方案

## 4.1 模型选择（按优先级排序）

核心原则：无需个人训练模型，利用通用大模型API+固定指令工程，即可实现稳定审核，优先选择国外模型保证严谨性，国内模型作为备选。

1. 首选：GPT-4/Gemini 1.5 Flash
        

    - 优势：严谨性强、不放水，理科逻辑（推导、计算）精准，格式服从度极高（仅输出标准化JSON），防敷衍、防抄录能力强，适合作为“无情审核官”。

    - 注意事项：需稳定外网环境，获取API Key，成本低（每日几次审核，月均几元），需注意账号风控。

2. 备选：国内模型（豆包API、DeepSeek-R1、讯飞星火）
        

    - 优势：无需外网，账号稳定，对中文教材、考纲熟悉，成本极低，适合不想折腾外网的场景。

    - 不足：偶尔会心软放水，理科逻辑精度略低于国外模型，格式服从度稍差（需通过指令强化）。

3. 不推荐：本地开源小模型（Llama 2/3、Qwen 7B等）
        

    - 劣势：易胡说、出题超纲、判分随意，部署麻烦，可靠性不足，不符合“强制审核”需求。

## 4.2 AI审核固定指令（核心，写死进程序）

指令核心：强制AI保持“无感情、标准化、严格审核”，输出固定JSON格式，便于程序解析，杜绝放水和自由发挥。

```text
你是一个严格、无感情、标准化的学习日活自动审核系统。
用户是高三学生，你只根据他提交的学习内容进行审核，绝不放水，绝不闲聊。

一、审核标准（必须严格执行）
1. 只有真正有效学习才算通过：
   - 有自己的理解、总结、推导、例题解答
   - 不是单纯抄书、抄题目、复制粘贴
   - 不是敷衍乱写、无关内容
2. 如果内容敷衍、空白、明显未思考，直接判定不通过。

二、出题规则
1. 必须基于用户提交内容出题，绝对不超纲
2. 只出基础题，用于检验是否真的学过
3. 固定出 3 道题
4. 题型：简答/判断/填空/选择均可
5. 题目必须简短、清晰、可快速回答

三、判分规则
1. 答对 2 题及以上 → 审核通过
2. 答对 1 题或全错 → 审核不通过
3. 判分严格，不模糊、不人情

四、输出格式（必须严格遵守，不得自由发挥）
请只返回JSON，不要其他任何文字：
{
  "审核结果": "通过"/"不通过",
  "理由": "简要说明",
  "题目": [
    "题目1",
    "题目2",
    "题目3"
  ],
  "正确答案": [
    "答案1",
    "答案2",
    "答案3"
  ]
}
```

## 4.3 审核流程

1. 用户提交学习成果（文字/图片），程序将内容转换为文本（图片通过OCR识别）。

2. 程序调用大模型API，传入上述固定指令+学习内容文本。

3. AI按规则完成审核、出题、判分，返回标准化JSON。

4. 程序解析JSON，判断审核结果：通过则更新授权状态，生成一次性动态密码（仅iOS端）；未通过则保持未授权状态，触发强制限制。

# 五、项目解读

## 5.1 项目核心优势（与现有自律工具对比）

|对比维度|现有工具（锁机APP、手动打卡）|本项目系统|
|---|---|---|
|强制力|弱，用户可手动关闭、卸载、绕开|强，系统级/高摩擦限制，用户无法绕过|
|智能判断|无，仅判断“是否打卡/是否锁屏”，无法识别假学习|有，AI审核学习真实性，出题验证理解程度|
|用户体验|差，完成学习后仍需手动解锁，操作繁琐|好，完成学习后无感解锁，未完成时高摩擦拦截|
|依赖他人|部分工具需要他人监督，易产生争端、浪费时间|无需他人介入，AI+程序全自动执行|
|防作弊能力|弱，可摆拍、抄录、敷衍打卡|强，AI识别敷衍/抄录，动态密码防冲动破戒|
## 5.2 项目可行性分析

### 5.2.1 技术可行性

个人完全可实现，无需专业开发经验，核心技术栈（Python、快捷指令、大模型API）均有成熟教程和工具，无需复杂设备、无需企业资质、无需上架应用商店，低成本快速落地。

- 后端：Python + 大模型API，核心代码量少，可直接复用模板。

- Windows端：Python + pywin32，调用系统API，无需复杂开发。

- Android端：Kotlin/Java，依托设备管理员权限，可直接编译APK。

- iOS端：快捷指令（自带工具），无需写复杂APP，导入逻辑即可使用。

### 5.2.2 成本可行性

成本极低，主要成本为大模型API调用费用，每日几次审核，月均花费几元（GPT-3.5/Gemini 1.5 Flash性价比极高）；无其他硬件、软件成本，现有电脑、手机即可满足需求。

### 5.2.3 落地难度

低，可分阶段落地：

1. 第一阶段（1天）：搭建AI审核模块（Python + 大模型API），实现“学习提交→AI审核→判分”。

2. 第二阶段（1-2天）：实现Windows/Android端基础强制锁死功能，完成核心闭环。

3. 第三阶段（1天）：配置iOS端快捷指令+动态密码，实现多端同步。

## 5.3 潜在问题与解决方案

1. 问题1：国外大模型API调用需外网，账号易风控。
        

    - 解决方案：备用国内模型（豆包API），无需外网；API Key妥善保管，避免异常调用；使用低价模型（如Gemini 1.5 Flash）降低成本。

2. 问题2：iOS端快捷指令被手动关闭，导致拦截失效。
        

    - 解决方案：限制快捷指令卸载（需输入设备密码）；配合屏幕使用时间双重限制；将快捷指令藏深，减少误触关闭概率。

3. 问题3：程序被手动杀进程、卸载，导致强制失效。
        

    - 解决方案：Windows端设为开机自启、管理员权限运行、防杀进程；Android端激活设备管理员，防止普通卸载；程序后台保活，异常退出后自动重启。

4. 问题4：AI审核误判（如误判认真学习为敷衍）。
        

    - 解决方案：优化AI指令，明确审核标准；增加“重新审核”功能，用户可提交补充学习内容，重新触发AI审核；选择精度更高的模型（如GPT-4）。

## 5.4 项目价值与意义

本项目并非传统的“自律技巧”，而是一套“技术驱动的强制自律系统”，核心价值在于：

- 解决个人自律的核心痛点：通过“程序强制+AI审核”，彻底杜绝“监守自盗”，让自律从“靠意志”变成“靠机制”。

- 模拟游戏官方机制，将学习转化为“日活任务”，通过“完成任务→解锁奖励”的闭环，提升学习主动性，减少拖延。

- 无他人介入，避免人情麻烦和争端，兼顾隐私与实用性，适合喜欢独立、不喜欢依赖他人的用户。

- 可复用、可优化，后续可增加“学习数据统计”“连续打卡奖励”等功能，进一步强化自律效果，贴合用户长期学习需求。

# 六、项目落地步骤（极简版）

1. 准备阶段（1小时）：获取大模型API Key（GPT-4/Gemini/豆包），安装Python环境，熟悉iOS快捷指令基础操作。

2. 第一阶段：搭建AI审核模块（1天）
        

    - 编写Python代码，调用大模型API，嵌入固定审核指令。

    - 测试审核逻辑：提交真实学习内容、敷衍内容，验证AI判分准确性和格式规范性。

3. 第二阶段：实现Windows/Android端强制锁死（1-2天）
        

    - Windows端：编写Python代码，调用系统API，实现锁屏、快捷键屏蔽、进程限制。

    - Android端：编写基础APK，申请设备管理员权限，实现强制锁屏、APP限制。

    - 联动AI审核模块，实现“审核通过→解锁”“未通过→锁死”的核心逻辑。

4. 第三阶段：配置iOS端快捷指令+动态密码（1天）


    - 创建快捷指令，配置状态变量、自动化触发、条件分支。
